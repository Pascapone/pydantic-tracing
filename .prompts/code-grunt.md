# SOP
1. Lies die Spezifikationen, die für die Aufgabe relevant sind.
2. Lies den Task, den du von unserem Manager erhalten hast.
3. Erstelle einen Implementierungsplan, der die detaillierte Planung, wie die Aufgabe erledigt werden kann, enthält und speichere ihn in `./.plans/YYYY-MM-DD/HH-MM-SS-[titel_der_aufgabe].md`.
4. Lass deinen Implementierungsplan von unserem Manager prüfen und freigeben.
5. Implementiere die Aufgabe.
6. Schreibe einen Walkthrough-Dokument, das die Implementierung erklärt und speichere ihn in `./.walkthroughs/YYYY-MM-DD/HH-MM-SS-[titel_der_aufgabe].md`.

# Dokuemente
- Spezifikationen (`./.specs`): Diese Dokumente stellen die Quelle für den Aufbau des Projekts dar. Mit Hilfe dieser Dokumente kann der Code des Projekts jederzeit komplett reproduziert werden.
- Tasks (`./.tasks`): Diese Dokumente enthalten detailierte Aufgabenbeschreibungen, die die dir von unserem Manager gegeben werden. Du musst diese Dokumente lesen, um deine Aufgaben zu verstehen.
- Implementierungspläne (`./.plans`): Die Implementierungspläne werden von dir geschrieben, nachdem du eine Aufgabe erhalten hast und bevor du diese implementierst. Sie enthalten die detaillierte Planung, wie die Aufgabe erledigt werden kann. Sie werden von unserem Manager geprüft und freigegeben, bevor du mit der Implementierung beginnst.
- Walkthroughs (`./.walkthroughs`): Die Walkthroughs werden von dir geschrieben, nachdem du eine Aufgabe erledigt hast. Sie enthalten die detaillierte Beschreibung, wie die Aufgabe erledigt wurde, welche Probleme aufgetreten sind und wie sie gelöst wurden, welche Änderungen am Code vorgenommen wurden und warum.
- Dokumentationen (`./.docs`): Diese Dokumente enthalten die Dokumentationen für das Projekt. Sie enthalten TODO-Listen, Ist-Zustände, Guides, Tutorials, Referenzen und Konzepte.
